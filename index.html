Browser tools

Browser tools have come a long way in the past few years, and they're pretty powerful tools now. Using the native tools built into your browser, you can test how your site performs on differently-sized screens, make quick changes to code and test them live, measure your site's performance, and debug issues more quickly.

Local development environments

If you want to really speed up your development time, you'll want to have a way to develop your sites locally. Investing some time in getting your local development

Benefits of developing your sites locally:

  • you're able to test all your changes before deploying them to your live production site
  • you can build a website when you don't have an internet connection, or when you have a poor internet connection
  • working locally is always faster, so you can make and test your changes more rapidly

What you need for your development environment will depend on what kind of site you're building and your development workflow. Ideally you want your local development environment to match your web server as closely as possible, so you'll be able to test in a similar environment.

Many people start with a local server stack, which is a server that runs on your local machine. Common configurations are LAMP (Linux Apache MySQL PHP) WAMP (Windows Apache MySQL PHP) or MAMP (Mac Apache MySQL PHP), since this is also what many web servers use.

You can also use a virtual machine. A virtual machine basically is an isolated part of your computer that acts as an independent computer. This means it has its own operating system that might be completely different from your computer's operating system. This can be helpful if you have a specific environment you'd like to test, but it can also eat up a lot of your computer's resources. VirtualBox is a popular open-source virtual machine, and you can use a provisioning tool like Vagrant to configure your virtual machine,

You may also want to try specialised tools like Docker or Laravel Valet. It all depends on what you're looking for!

Starter themes & frameworks

Starter themes are a great way to get started building a new site quickly. No matter what CMS you use, there's a starter theme out there for you. Starter themes are usually relatively style-agnostic, so you aren't tied to a particular look and feel. They cover a lot of the basics, so you can jump into the fun part of customisation.

What kind of basics? Well, all starter themes will include, at the very least, your basic HTML markup. They'll usually also contain all the special templates your CMS needs in a theme, as well as some special functionality to make your themeing experience easier.

Some will also have some CSS included: various resets and accessibility standards, sometimes some sensible defaults to get you started. Some will come with other tools like a build setup or sass sources.

What about frameworks?

Frameworks are a little bit different, because they often are quite opinionated about style. Depending on who's talking, a framework might be a full theme framework (like Genesis) or a CSS framework (like Bootstrap or Foundation). These can be great tools if you don't want a lot of control, or you want things to look good out of the box. If you want to customise things a bit more, you'll want to stick to starter themes.

Git & Github

Git is an open-source version control tool. It allows you to track every change to your code so you can easily roll back to prior versions when needed. Using a version control system makes it easier to collaborate with other contributors and means that you'll never lose your work again.

Unlike other popular version control software, git is distributed, which means that instead of having a single place where version history is saved, each working copy of the code also has the full version history. This makes git much faster, and far easier to use on a poor or nonexistent internet connection.

Github is a popular place to host your git repository, and it's extremely popular with maintainers of open-source projects. If you're contributing to open-source, you'll probably come across Github!

CSS Precompilers

Have you ever changed your mind in the middle of building a site and suddenly wanting to change all the colours? With a CSS precompiler, that's a thing you can do!

Pre-processors extend CSS with variables, operators, interpolations, functions, and mixins in order to help you write more CSS code in less time. Learning to use them if you already know CSS is pretty simple, since they tend to follow a similar or identical structure to CSS. SCSS in particular is a great introduction, because you can write vanilla CSS as well.

In order to use a CSS precompiler on your website, you'll need to compile your files: basically convert your code into CSS code. There are many automated tools that can do this for you, like Koala. You can also write SCSS directly in your WordPress admin using the Jetpack plugin, or you can use a task runner to compile your code as well as automating other tasks.

Task Runners

Task runners are command-line tools that take care of some of the automated tasks involved in building a website. Automating these tasks means you can worry about the important parts of building a site.

Things task runners can do for you:

  • Compile CSS from your preprocessed source files.
  • Add browser prefixes to CSS rules.
  • Create SVG sprites for an icon system, and PNG fallbacks.
  • Concatenante JS files.
  • Lint your code for style.
  • Optimise images.
  • Refresh your browser when code changes.
